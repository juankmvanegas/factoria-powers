# /qa-plan

Creates or updates the QA test plan for a feature, migration, or release.

## What it does
Translates the QA strategy into an actionable plan with suites, environments, dependencies, execution order, and gates.

## Instructions
1. Read `.cloud/qa/strategy/qa-strategy.md`
2. Enumerate required suites and environments
3. Document execution order, dependencies, and blocking conditions
4. Update `.cloud/qa/plans/test-plan.md`

## Usage
```text
/qa-plan [feature, module, release]
```
