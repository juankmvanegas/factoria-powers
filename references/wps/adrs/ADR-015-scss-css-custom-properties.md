# ADR-015: SCSS with CSS Custom Properties

## Status
Accepted

## Date
2024-04-18 (Factoria-Wps v1.0.0)

## Context
The project needs a styling approach that supports both the build toolchain (SCSS compilation) and runtime theming (CSS custom properties from theme.json).

## Decision
Use SCSS for block and component styles combined with CSS custom properties for theming:

### SCSS Usage
- Block styles in `editor.scss` and `style.scss`
- Component styles in component directories
- SCSS features: nesting, variables, mixins, imports

### CSS Custom Properties
Two sources of custom properties:

#### 1. Theme Root Variables (style.css)
```css
:root {
  --navyBlue: #001a4a;
  --blue: #00369c;
  --green: #3ec92b;
  --aqua: #65baaf;
  --white: #fff;
}
```

#### 2. WordPress Generated (theme.json)
```css
/* Auto-generated by WordPress from theme.json */
--wp--preset--color--primary: #00369C;
--wp--preset--color--secondary: #3EC92B;
--wp--preset--spacing--20: 0.44rem;
```

### Priority Order
1. Use `var(--wp--preset--color--{slug})` for theme.json-defined values
2. Use `var(--{name})` for custom root variables
3. Use SCSS variables (`$var`) for build-time-only values (breakpoints, z-indexes)

### Standard Breakpoints
```scss
$bp-tablet: 726px;
$bp-desktop: 1024px;

@media (min-width: $bp-tablet) { /* Tablet styles */ }
@media (min-width: $bp-desktop) { /* Desktop styles */ }
```

### Style File Organization
| File | Scope | When to Use |
|------|-------|------------|
| `editor.scss` | Editor only | Block outlines, placeholders, admin-only UI |
| `style.scss` | Editor + frontend | Visual design, layout, typography |
| `global.css` | All blocks | Shared utilities and resets |

## Consequences
- Runtime theme changes via CSS custom properties
- Build-time optimization via SCSS compilation
- Consistent color usage across blocks and components
- WordPress theme.json palette visible in block color pickers
- Two variable systems to manage (root + WordPress preset)
