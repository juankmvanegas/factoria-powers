# Skill: Database (Auto-Activated)

## Activation Trigger

This skill activates automatically when database models, repositories, or migrations are touched.

## Purpose

Enforce correct usage of SQLAlchemy 2.0 async patterns, Alembic migrations, and the repository pattern aligned with Clean Architecture.

## Enforcement Rules

### SQLAlchemy 2.0 Async Patterns

- Use `AsyncSession` for all database operations
- Use `select()` statements (2.0 style) â€” NEVER use legacy `Query` API
- Use `Mapped[]` type annotations on all model attributes
- Use `mapped_column()` instead of `Column()`
- Use `relationship()` with `Mapped[]` type hints

### Declarative Models

```python
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column, relationship
from sqlalchemy import String, ForeignKey
from datetime import datetime
from uuid import UUID

class Base(DeclarativeBase):
    pass

class OrderModel(Base):
    __tablename__ = "orders"

    id: Mapped[UUID] = mapped_column(primary_key=True)
    customer_id: Mapped[UUID] = mapped_column(ForeignKey("customers.id"))
    status: Mapped[str] = mapped_column(String(50))
    created_at: Mapped[datetime] = mapped_column(default=datetime.utcnow)
    items: Mapped[list["OrderItemModel"]] = relationship(back_populates="order")
```

- ALL models MUST inherit from the shared `Base`
- Use UUID primary keys (not auto-increment integers)
- Include `created_at` and `updated_at` audit columns
- Use meaningful `__tablename__` (plural, snake_case)

### Repository Pattern

- Define repository ports as `Protocol` in `application/ports/`
- Implement repository adapters in `infrastructure/persistence/`
- Repositories accept `AsyncSession` via constructor injection
- NEVER expose SQLAlchemy models outside Infrastructure layer â€” map to Domain entities

```python
# application/ports/order_repository.py
class OrderRepository(Protocol):
    async def get_by_id(self, order_id: UUID) -> Order | None: ...
    async def save(self, order: Order) -> Order: ...
    async def delete(self, order_id: UUID) -> None: ...

# infrastructure/persistence/sqlalchemy_order_repository.py
class SqlAlchemyOrderRepository:
    def __init__(self, session: AsyncSession) -> None:
        self._session = session

    async def get_by_id(self, order_id: UUID) -> Order | None:
        stmt = select(OrderModel).where(OrderModel.id == order_id)
        result = await self._session.execute(stmt)
        model = result.scalar_one_or_none()
        return self._to_domain(model) if model else None
```

### Alembic Migration Rules

- Autogenerate migrations with `alembic revision --autogenerate -m "description"`
- Review autogenerated migrations before applying
- EVERY migration MUST have both `upgrade()` and `downgrade()`
- Use descriptive migration messages (e.g., "add_status_column_to_orders")
- NEVER edit a migration that has been applied to any environment
- For data migrations, create separate migration files from schema migrations

### AsyncSession Management

- Use a session factory (`async_sessionmaker`) configured at startup
- Inject sessions via FastAPI `Depends()` â€” NEVER create sessions manually in business code
- Use context managers for session lifecycle

```python
async def get_session() -> AsyncGenerator[AsyncSession, None]:
    async with async_session_factory() as session:
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()
            raise
```

### Unit of Work Pattern

- Group related repository operations in a single transaction
- Commit once at the end of the use case, not per repository call
- Rollback on any exception
- NEVER nest transactions â€” use savepoints if needed (`session.begin_nested()`)

### Connection Pooling

- Configure pool size based on expected concurrency
- Set `pool_pre_ping=True` to handle stale connections
- Use `pool_recycle` for long-lived applications
- Monitor pool usage in production

### Parameterized Queries

- NEVER use string formatting or f-strings in SQL queries
- NEVER use `.format()` or `%` operator for SQL
- ALWAYS use SQLAlchemy's parameter binding or `text()` with `:param` syntax
- This applies to ALL SQL, including raw queries via `session.execute(text(...))`
