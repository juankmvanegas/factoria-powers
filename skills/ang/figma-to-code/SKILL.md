# Skill: Figma to Code

## Purpose

Receive a design from any source (Figma, Google Stitch, image) and convert it into Angular components respecting Clean Architecture 3 layers, ITCSS, and all Factoria-Ang conventions. The agent **SEES** the design and generates pixel-faithful code.

## Input Methods (from highest to lowest fidelity)

### Method 1: Figma MCP (Recommended)

The agent connects directly to Figma via MCP and reads the complete design: components, colors, typography, spacing, assets. **No screenshots needed.**

**Setup (one time only):**
```bash
# Option A: Remote MCP (recommended)
claude mcp add --transport http figma https://mcp.figma.com/mcp

# Option B: For all projects
claude mcp add --scope user --transport http figma https://mcp.figma.com/mcp
```

**Authentication:**
1. In Claude Code, type `/mcp`
2. Select `figma`
3. Click "Authenticate" → "Allow Access"

**Usage:**
The user pastes the Figma frame/component URL:
```
"Implement this design: https://www.figma.com/design/ABC123/ProjectName?node-id=1234"
```

The agent:
1. Reads the design via MCP (structure, colors, types, spacing)
2. Extracts design tokens
3. Generates Angular components faithful to the design

### Method 2: Image on Disk (when there is no MCP)

> **NOTE**: Claude Code runs in the terminal — images cannot be pasted directly.
> The dev exports from Figma and saves to disk. Claude Code reads the file.

**Flow:**
1. In Figma: right-click → Export → PNG (or use Ctrl+Shift+E)
2. Save to a project or temporary folder (e.g., `D:/designs/`)
3. Pass the **file path** to the agent:

```
"Implement this design: D:/designs/notes-view.png"
```

```
"Implement this design: ./designs/home-dashboard.png"
```

**Tip**: In many terminals you can **drag the file** to the window and the path is pasted automatically.

The agent:
1. Reads the image from disk (Claude Code is multimodal — it sees images)
2. Analyzes layout, colors, typography, components
3. Generates Angular code based on the visual
4. Uses Playwright to capture a screenshot of the result
5. Visually compares with the original design → auto-corrects

**For multiple screens**, export each one:
```
"Implement these views:
- Home: D:/designs/home.png
- Notes list: D:/designs/notes-list.png
- Note detail: D:/designs/note-detail.png"
```

### Method 3: Figma Dev Mode — Manual Specs

The user copies specs from Figma Dev Mode and pastes them as text:
```
"Implement this component:
- Card of 320x200px
- Background: #FFFFFF
- Border radius: 8px
- Shadow: 0 2px 8px rgba(0,0,0,0.1)
- Title: Inter 16px semibold #1A1A1A
- Description: Inter 14px regular #666666
- Padding: 16px
- Gap between elements: 8px"
```

### Method 4: Google Stitch MCP

Google Stitch generates UIs with AI and exports them as HTML/CSS. The agent reads the Stitch designs, extracts the generated HTML, and converts it to Angular components with ITCSS.

**Setup (one time only):**
```bash
# Initialize (auth wizard with Google Cloud)
npx @_davideast/stitch-mcp init

# Or with API key directly
# Set environment variable STITCH_API_KEY
```

**Add to Claude Code:**
```bash
claude mcp add stitch -- npx @_davideast/stitch-mcp proxy
```

**Or in the project's `.mcp.json`:**
```json
{
  "mcpServers": {
    "stitch": {
      "command": "npx",
      "args": ["@_davideast/stitch-mcp", "proxy"]
    }
  }
}
```

**Usage:**
```
"Convert this Stitch project to Angular: [project-id or Stitch URL]"
```

**Available tools via MCP:**
- `build_site` — Builds a site mapping screens to routes, returns HTML for each page
- `get_screen_code` — Gets the HTML/CSS of a specific screen
- `get_screen_image` — Gets a screenshot of the screen as an image (base64)

**The agent:**
1. Reads the Stitch project screens via MCP
2. Gets the HTML/CSS generated by Stitch
3. Converts static HTML to Angular components (inline templates)
4. Maps CSS styles to ITCSS (variables, objects, components)
5. Creates the views/pages/components structure
6. Validates with Playwright

**Additional CLI (without MCP):**
```bash
# Serve all screens locally for inspection
npx @_davideast/stitch-mcp serve -p {project-id}

# Build complete site with routes
npx @_davideast/stitch-mcp site -p {project-id}
```

### Method 5: Combined (Maximum fidelity)

Combine multiple sources for the best result:

```
"Implement this design:
- Figma (exact tokens): https://www.figma.com/design/ABC/Project?node-id=123
- Stitch (HTML reference): [Stitch project-id]
- Visual (validation): D:/designs/notes-mockup.png"
```

The agent uses Figma MCP for exact design tokens, Stitch for reference HTML/CSS, and the image for final visual validation.

## When to use each method

| Method | Best for | Fidelity | Setup |
|--------|----------|----------|-------|
| **Figma MCP** | Team with established Figma | Very high | `claude mcp add figma` |
| **Stitch MCP** | Rapid prototyping with AI, ready HTML | High | `npx stitch-mcp init` |
| **Image on disk** | No MCP, exported designs | Medium-High | None |
| **Manual specs** | Specific adjustments, no image | Medium | None |
| **Combined** | Production, pixel-perfect | Maximum | Both MCPs |

## Execution Flow

### Phase 1: Capture the Design

1. **With MCP**: Read the design from Figma directly
2. **With Screenshot**: Analyze the provided image
3. **With Specs**: Parse user specifications

Extract:
- **Layout**: Structure, grids, flexbox, spacing
- **Typography**: Fonts, sizes, weights, line-heights
- **Colors**: Complete palette (background, text, borders, accents)
- **Components**: Buttons, inputs, cards, tables, modals
- **Iconography**: Icons used
- **States**: Hover, active, disabled, error, loading
- **Responsive**: Breakpoints if defined

### Phase 2: Map to ITCSS

Classify each visual element into the correct ITCSS layer:

| Design Element | ITCSS Layer | File |
|----------------|-------------|------|
| Brand colors | _settings/ | `_colors.scss` |
| Base typography | _settings/ | `_typography.scss` |
| Spacing scale | _settings/ | `_spacing.scss` |
| Layout mixins | _tools/ | `_mixins.scss` |
| Body, h1-h6 styles | _elements/ | `_headings.scss` |
| Grid/layout | _objects/ | `_layout-*.scss` |
| Cards, buttons, forms | _components/ | `_card.scss`, `_button.scss` |
| Utilities | _trumps/ | `_utilities.scss` |

### Phase 3: Map to Clean Architecture

| Design Element | Angular Layer | Type |
|----------------|--------------|------|
| Full screen/page | Presentation/views/ | Page |
| Reusable section | Presentation/views/components/ | Component |
| Generic UI component | Presentation/common/static-components/ | Shared Component |
| Modal/Dialog | Presentation/common/dynamic-components/ | Dynamic Component |
| Design data | Application/dtos/ | DTO |
| Interaction logic | Application/services/ | Service |
| API calls | Infrastructure/services/ | Adapter |

### Phase 4: Generate Code

Follow the Factoria-Ang execution order:

1. **Application** (if there is new logic/data)
   - DTOs for data displayed in the UI
   - Use Cases if there are new interactions
   - Services if there is new logic

2. **Infrastructure** (if there are new APIs)
   - HTTP Adapters for new endpoints

3. **Presentation** (always)
   - ITCSS styles (settings → components)
   - Angular components (inline template)
   - Pages/Views
   - Routing if it is a new view

### Phase 5: Visual Validation

**With Playwright** (mandatory for designs):

```bash
# Serve the app
ng serve &

# Capture screenshot of the result
npx playwright screenshot http://localhost:4200/{route} \
  --output .qa-reports/figma-validation/result.png
```

The agent compares visually:
- Do the colors match?
- Is the spacing correct?
- Is the typography faithful?
- Does the layout respond correctly?
- Do states (hover, active) work?

If it does NOT match → fix → screenshot → compare again.
Maximum 3 iterations.

### Phase 6: Design Tokens (Optional but Recommended)

If the Figma design has defined design tokens, extract them and map to SCSS variables:

```scss
// _settings/_design-tokens.scss
$color-primary: #2563EB;
$color-secondary: #7C3AED;
$color-success: #10B981;
$color-error: #EF4444;
$color-text-primary: #1A1A1A;
$color-text-secondary: #666666;
$color-background: #FFFFFF;
$color-surface: #F9FAFB;

$font-family-primary: 'Inter', sans-serif;
$font-size-xs: 12px;
$font-size-sm: 14px;
$font-size-md: 16px;
$font-size-lg: 20px;
$font-size-xl: 24px;

$spacing-xs: 4px;
$spacing-sm: 8px;
$spacing-md: 16px;
$spacing-lg: 24px;
$spacing-xl: 32px;

$border-radius-sm: 4px;
$border-radius-md: 8px;
$border-radius-lg: 12px;
```

```

1. [Reads image from disk]
   → Detects: Card list with header, filters, paginated table
```

## Rules

- ALWAYS respect Figma design tokens (exact colors, types, spacing)
- ALWAYS map to ITCSS — no inline styles or :host with hardcoded values
- ALWAYS visually validate with Playwright after generating
- ALWAYS use inline templates
- NEVER invent colors or spacing — use exactly what the design says
- NEVER create components outside Presentation
- If the design does not specify a state (hover, error), ask the user
- If the design uses a component that already exists, REUSE — do not duplicate
- Maximum 3 visual auto-correction iterations before escalating
